## Ringkasan Cepat

Sistem giliran klinik sesuai untuk projek web kerana aliran pengguna jelas. Pesakit daftar, sistem beri nombor giliran, staf panggil giliran, dan admin semak laporan.

## Skop Projek

Bina web app dengan pendaftaran, paparan nombor giliran, panel staf, status selesai atau batal, dan laporan harian. Firebase sesuai untuk authentication, database dan kemas kini masa nyata.

## Modul Dicadangkan

Masukkan modul daftar pesakit, jana tiket, panel counter, paparan giliran, pengumuman, laporan admin dan audit log. Gunakan role admin dan staf.

## Struktur Data

Sediakan koleksi untuk users, queue tickets, counters, announcements dan audit logs. Rekod giliran perlu ada nombor, status, masa daftar, masa dipanggil dan masa selesai.

## Validasi Demo

Tunjuk aliran penuh daripada daftar hingga selesai. Uji banyak pengguna, giliran dibatalkan, refresh page dan tindakan khas staf.

## Kesilapan Biasa

- Tiada pemisahan role.
- Nombor giliran reset dengan salah.
- Tiada timestamp untuk kira masa menunggu.
- Simpan data kesihatan yang tidak perlu.
- Interface siap tetapi workflow tidak lengkap.
